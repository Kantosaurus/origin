## Design Context

### Users
Developers who spend hours daily in a terminal coding harness. They're using
origin as their primary AI coding tool — an alternative to Claude Code, jcode,
and opencode. They care about speed, clarity, and feeling in control. The TUI
is their workspace; it needs to feel like a precision instrument, not a toy.

### Brand Personality
**Precise, assured, distinctive.** Origin should feel like a tool built by
someone with strong opinions about craft. Not flashy, not timid — confident.
The kind of tool where every visual choice looks intentional.

### Aesthetic Direction
- **Tone**: Polished & premium — the Claude Code screenshot is the quality
  bar, but origin should have its own visual identity, not a clone.
- **Theme**: Dark. Developers in terminals are in dark environments; fighting
  that is fighting the context.
- **Palette**: Origin's own — distinct from Claude Code's blue/purple, jcode's
  defaults, and opencode's green. Choose a palette that feels uniquely origin.
- **Anti-references**: Generic ANSI-16 color terminals. Flat gray-on-black with
  no hierarchy. The "default htop" look.

### Design Principles
1. **Information density without noise** — Show everything the developer needs,
   nothing they don't. Every visual element earns its space.
2. **Color as meaning** — Colors communicate state (diffs, errors, tool
   activity), not decoration. Muted surfaces, purposeful accents.
3. **Typographic hierarchy in a monospace world** — Terminals are monospace,
   but brightness, weight (bold), and spacing create real hierarchy.
4. **Consistency across all states** — Streaming, idle, error, tool dispatch,
   diff view — every state should feel like it belongs to the same system.
5. **Distinctive identity** — Someone glancing at the screen should recognize
   origin, not mistake it for Claude Code or any other tool.
